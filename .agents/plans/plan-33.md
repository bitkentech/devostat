# Plan 33: Replace SessionStart hook with cross-platform Node.js bootstrap + generate hooks.json via Jackson

## Context

`plugin-resources/src/main/resources/hooks/hooks.json` has a `"command"` field containing
~600 chars of semicolon-chained bash on a single line. It is unreadable, untestable directly,
requires Maven `@...@` filtering, and is POSIX-only (breaks on Windows).

`plugin-node` already ships compiled JS to users and Node is guaranteed available in Claude Code's
hook environment. The right fix is to replace the shell logic entirely with a JS-based approach
that works on Windows, macOS, and Linux without any shell utilities.

Three improvements:
1. **Write `session-start.ts`** in `plugin-node/src/` containing the jlink install logic in JS
   (`fs`, `child_process`, `https`). Compiled to `dist/session-start.js` alongside the other task scripts.
2. **Replace the hooks.json command** with a single `node -e "..."` bootstrap that copies `dist/`
   from the plugin root to cache and then requires `session-start.js` — no shell utilities,
   no `~/` expansion, no `mkdir -p`, no `cp`.
3. **Generate hooks.json via Jackson** in `ResourceBuilder` (plugin-skill). Maven filtering removed.

**Backlog issue:** Build-infrastructure improvement. Tracking locally only.

---

## Design

### Hook command (generated by Jackson, baked at build time)

```
node -e "const o=require('os'),fs=require('fs'),p=require('path');const c=p.join(o.homedir(),'.cache','shipsmooth-dev');fs.cpSync(p.join(process.env.CLAUDE_PLUGIN_ROOT,'dist'),p.join(c,'dist'),{recursive:true,force:true});require(p.join(c,'dist','session-start.js'))"
```

- `fs.cpSync` — cross-platform recursive copy (Node 16.7+), no shell needed
- `os.homedir()` — resolves home dir on all platforms (no `~/` expansion)
- `process.env.CLAUDE_PLUGIN_ROOT` — set by Claude Code on all platforms
- `require(session-start.js)` — runs the install logic in-process after the copy
- The cache dir path (`shipsmooth-dev`) is baked in by Jackson from `PluginModel.cacheDir()`

### session-start.ts responsibilities

Reads `CLAUDE_PLUGIN_ROOT` and `cacheDir` (hardcoded from build time via a companion
`session-start-config.json` or constants baked into the JS at build time). Performs:
- Check if `{cacheDir}/runtime-{version}/bin/shipsmooth-tasks` is executable — skip if so.
- Detect platform via `process.platform` / `process.arch` (no `uname`).
- If `{CLAUDE_PLUGIN_ROOT}/runtime` exists → `fs.cpSync` it to cache, `fs.chmodSync` the binary.
- Else → download zip from GitHub releases via Node `https`, extract via `child_process.execFileSync('unzip', ...)`
  or a pure-JS zip library if available.
- Print status to stdout.

### Module responsibilities after the change

| Output file | Who writes it | Phase |
|---|---|---|
| `build/hooks/hooks.json` | plugin-skill (Jackson via ResourceBuilder) | compile |
| `build/dist/*.js` | plugin-node (tsc) + plugin-dist copy | process-resources |
| `build/runtime/` | plugin-dist dev profile (copy jlink image) | process-resources |

Note: `session-start.sh` created in the de-risk draft is **removed** — superseded by `session-start.ts`.

## Tasks

### Task 1: Write session-start.ts in plugin-node [High]

Create `plugin-node/src/main/scripts/tasks/session-start.ts`.

Uses only Node built-ins (`fs`, `os`, `path`, `https`, `child_process`). No new npm dependencies.

Key decisions:
- Version and cache dir are not available as env vars at runtime — they must be baked into
  the JS at build time. Use a separate `session-start-config.json` written to `build/dist/`
  by ResourceBuilder (alongside hooks.json), containing `{ "version": "0.2.0", "cacheDir": "~/.cache/shipsmooth-dev" }`.
  `session-start.js` reads this sibling file via `path.join(__dirname, 'session-start-config.json')`.
- Platform detection: `process.platform === 'linux'` and `process.arch === 'x64'` (no `uname`).
- Zip extraction: use `child_process.execFileSync('unzip', [...])` — `unzip` is available on
  Linux/macOS; Windows support deferred (exit with clear message for now, same as current behavior).
- `fs.chmodSync(bin, 0o755)` replaces `chmod 755`.
- Atomic install: copy to `{runtimeDir}.tmp` first, then `fs.renameSync` to final path.

Tests: add `session-start.test.ts` in `plugin-node/src/test/` covering:
- Already-cached binary → no-op
- Local `CLAUDE_PLUGIN_ROOT/runtime` exists → copies to cache
- Unsupported platform → throws/exits with message

### Task 2: Write session-start-config.json generation in ResourceBuilder [Low]

In `ResourceBuilder.java`, after writing `hooks.json`, also write `build/dist/session-start-config.json`:

```java
ObjectNode config = mapper.createObjectNode()
    .put("version", model.pluginVersion())
    .put("cacheDir", model.cacheDir());
mapper.writerWithDefaultPrettyPrinter()
    .writeValue(distDir.resolve("session-start-config.json").toFile(), config);
```

`distDir` = `Path.of(buildOutputDir, "dist")` — same dir where `plugin-dist` copies the compiled JS files.

### Task 3: Generate hooks.json via Jackson in ResourceBuilder [Low]

Add `jackson-databind` dependency to `plugin-skill/pom.xml`. In `ResourceBuilder.java`, build and
write `build/hooks/hooks.json` programmatically:

```java
String bootstrap = "const o=require('os'),fs=require('fs'),p=require('path');"
    + "const c=p.join(o.homedir(),'" + model.cacheDir().replace("~/.cache/", ".cache/") ... + "');"
    // ... full one-liner
```

Actually: the cache dir is an absolute-ish path like `~/.cache/shipsmooth-dev`. Since `os.homedir()`
replaces `~`, construct the cache path in the JS as:
`p.join(o.homedir(),'.cache','shipsmooth-dev')` where `shipsmooth-dev` is the last segment of
`model.cacheDir()` (extract via `Path.of(model.cacheDir()).getFileName()`).

Jackson serialises the string value — no manual JSON escaping needed.

### Task 7: Copy jlink image to build/runtime/ in plugin-dist dev profile [Low]

Add `maven-resources-plugin` execution in `plugin-dist/pom.xml` dev profile (after `verify-jlink-image-exists`):

```xml
<execution>
  <id>copy-jlink-runtime</id>
  <phase>process-resources</phase>
  <goals><goal>copy-resources</goal></goals>
  <configuration>
    <outputDirectory>${build.outputDir}/runtime</outputDirectory>
    <resources>
      <resource>
        <directory>${shipsmooth.jlink.dir}</directory>
        <filtering>false</filtering>
      </resource>
    </resources>
  </configuration>
</execution>
```
