# Plan 18 — Update org/marketplace identifiers (bitkentech rebrand)

## Context

The GitHub repo for this project is moving from `pramodbiligiri/devostat` to `bitkentech/devostat`, and the sibling marketplace repo is moving from `pramodbiligiri/claude-plugins` to `bitkentech/claude-plugins`. Additionally, the marketplace *name* (used in `/plugin install devostat@<marketplace>`) is being changed from `pramodb-plugins` to `bitkentech` so that the install identifier matches the owning org — making it easier for users to recognize the same identity across places.

This plan updates every repo-internal reference that must reflect the new identifiers. The sibling marketplace repo (`bitkentech/claude-plugins`) itself is out of scope — its `marketplace.json` will be updated separately when that repo is created/renamed.

Following the plan-15 pattern, URLs are being updated in anticipation of the rename (tracked as a separate action for the human to execute at the GitHub level). Historical plan files in `.agents/plans/` are left unchanged — they reflect identifiers at their time of writing. Session transcripts under `docs/session/` are likewise immutable.

## Scope

### In scope — three identifier changes
1. **Repo URL:** `pramodbiligiri/devostat` → `bitkentech/devostat`
2. **Marketplace repo:** `pramodbiligiri/claude-plugins` → `bitkentech/claude-plugins`
3. **Marketplace name:** `pramodb-plugins` → `bitkentech` (keeping the install command `devostat@bitkentech` aligned with the owning org)

### Out of scope
- Maven `groupId` values (`com.github.pramodbiligiri` in the four `pom.xml` files). Artifact coordinate — user explicitly declined changing this.
- `.agents/plans/plan-*.md` — immutable audit trail.
- `docs/session/*.jsonl` — immutable session transcripts.
- The marketplace.json inside the sibling `bitkentech/claude-plugins` repo — handled there, not here.
- `build/` output — regenerated by Maven from `plugin-resources/src/main/resources/`; do not hand-edit.
- Local cache path references in historical plans (e.g. `~/.claude/plugins/cache/pramodb-plugins/...`) — historical.

## Notes on the two `plugin.json` files

- `.claude-plugin/plugin.json` at the repo root is a standalone committed file (not generated).
- `plugin-resources/src/main/resources/claude-plugin/plugin.json` is filtered by `plugin-resources/pom.xml` (execution `copy-plugin-meta`) into `build/.claude-plugin/plugin.json`. The `build/` copy is gitignored.
- Both must be updated to stay in sync; `build/` regenerates automatically on `mvn process-resources`.
